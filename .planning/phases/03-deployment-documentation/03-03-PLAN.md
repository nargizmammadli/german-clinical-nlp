<objective>
Generate three pre-committed example extraction JSON files by running live Mistral 7B Q4_K_M inference against the running docker-compose API. These files allow portfolio reviewers to inspect real extraction quality, confidence scores, and source spans on the repo landing page without needing to run the model themselves.

Purpose: Completes the portfolio story — a reviewer who cannot download the 4.5GB model can still see authentic extraction output, understand confidence scoring, and verify source span grounding. This is the final piece that makes the repo self-demonstrating.

Output:
- data/examples/temporal_extraction.json — date-heavy German oncology note; extracted Date and LOS entities per D-11 Sample 1
- data/examples/diagnosis_extraction.json — ICD-coded diagnosis text; extracted Diagnosis entities per D-11 Sample 2
- data/examples/medication_los_extraction.json — mixed prescription + Verweildauer text; extracted Medication and LOS entities per D-11 Sample 3
All three files include _generated_by and _input_text metadata fields per D-12.
</objective>

       ```

       mkdir -p models
       wget -O models/mistral-7b-instruct.gguf \

       ```
       curl http://localhost:8000/health

       ```
       curl -X POST http://localhost:8000/extract \
         -H "Content-Type: application/json" \

       ```
       Expected: JSON response with temporal_entities array containing a Date entity.
