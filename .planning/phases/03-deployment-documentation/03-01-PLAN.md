<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host filesystem → container | Volume mount injects model file and .env secrets into container at runtime |
| Environment variables → src/config.py | MODEL_PATH injected by docker-compose is validated by config.py path traversal check |
| Container process → host | Non-root USER appuser limits blast radius if container is compromised |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | Docker container process | mitigate | `RUN adduser --disabled-password --gecos "" appuser` + `USER appuser` in Dockerfile (ASVS V1.6) |
| T-03-02 | Information Disclosure | .env secrets in image layers | mitigate | `.dockerignore` excludes `.env` and `.env.*`; env vars injected at runtime via `env_file` in docker-compose |
| T-03-03 | Tampering | GGUF model file from container | mitigate | Volume mount uses `:ro` (read-only) flag — container cannot modify model file on host |
| T-03-04 | Tampering | MODEL_PATH path traversal via env var | accept | Already mitigated in `src/config.py` T-01-01: validates MODEL_PATH.resolve().is_relative_to(PROJECT_ROOT / "models") |
| T-03-05 | Denial of Service | Unbounded memory consumption | mitigate | `mem_limit: 8g` in docker-compose enforces hard OOM kill threshold; prevents host memory exhaustion |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this phase beyond adding loguru (pre-verified HIGH confidence in CLAUDE.md) to existing pyproject.toml runtime deps |
</threat_model>
