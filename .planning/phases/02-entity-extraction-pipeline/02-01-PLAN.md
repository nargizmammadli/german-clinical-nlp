<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → /extract endpoint | Untrusted German clinical text crosses into LLM processing |
| LLM output → Pydantic validation | Model-generated JSON crosses into application data layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | POST /extract | mitigate | Validate input text non-empty (ExtractionRequest.text with min_length=1), validate text length < 8192 characters (context window limit), return 400 for invalid requests |
| T-02-02 | Information Disclosure | Error responses | mitigate | Per D-05/D-08: return structured errors array without stack traces, sanitize LLM output errors to remove model internals |
| T-02-03 | Tampering | LLM JSON output | mitigate | Pydantic validation enforces schema conformance, confidence range (0.0-1.0), source span validation per D-14/D-15 flags ungrounded entities |
| T-02-04 | Spoofing | Source span grounding | accept | LLM can claim false source spans - detected via D-14 exact match, flagged with source_span_validated=false, not rejected per D-15 (low risk for portfolio demo) |
</threat_model>
