# German Clinical Sample Data

## Overview

This directory contains sample German clinical text data used for demonstrating entity extraction capabilities in the German Clinical NLP Pipeline.

## Data Source

**Current:** Synthetic German Clinical Data (15 samples)

**Originally Planned:** GGPONC 2.0 dataset from HuggingFace (`bigbio/ggponc2`)

**Why Synthetic:** The GGPONC 2.0 dataset on HuggingFace uses deprecated loading scripts that are no longer supported by the `datasets` library (version 3.0+). Rather than downgrade dependencies, we generated realistic synthetic samples to demonstrate the same capabilities.

## Sample Structure

Each sample in `ggponc_samples.json` contains:

- **id**: Unique identifier (format: `synthetic_NNN`)
- **text**: German clinical text (discharge notes, consultation reports, etc.)
- **source**: Data source identifier ("Synthetic German Clinical Data")
- **entities**: Entity annotations with type, text, and character offsets

## Entity Types

The samples include annotations for common clinical entity types:

- **Date**: Dates in German format (DD.MM.YYYY)
- **Diagnosis**: ICD-10 coded diagnoses (e.g., "Diabetes mellitus Typ 2 (E11.9)")
- **Medication**: Drug names with dosages (e.g., "Metformin 1000mg")
- **Procedure**: Medical procedures (e.g., "Laparoskopische Cholezystektomie")
- **Symptom**: Clinical symptoms (e.g., "chronische Rückenschmerzen")
- **LOS**: Length-of-stay indicators (e.g., "4 Tage")
